# Contribuindo para o Pybase

Obrigado pelo interesse em contribuir! 🎉

## Como contribuir

1. **Fork** o repositório
2. Crie um **branch** para sua feature (`git checkout -b feat/minha-feature`)
3. Faça suas alterações
4. Execute os testes (`python -m pytest tests/`)
5. Commit com mensagem descritiva
6. Abra um **Pull Request**

## Padrões de código

- Python 3.10+
- Siga o estilo existente do código
- Sem dependências externas — use apenas stdlib
- Testes novos para funcionalidades novas
- Docstrings em português ou inglês

## Testes

```bash
# Rodar todos os testes
python -m pytest tests/ -v

# Rodar um arquivo específico
python -m pytest tests/test_database.py -v
```

## Pull Requests

- Título claro e descritivo
- Descreva o que muda e por quê
- Referencie issues relacionadas
- Mantenha o escopo focado (um PR = uma coisa)

## Reportando bugs

Abra uma [issue](https://github.com/marcosgabrielgomes110-collab/Pybase2/issues) com:

- Passos para reproduzir
- Comportamento esperado vs. real
- Versão do Python e ambiente

## Dúvidas

Abra uma [discussão](https://github.com/marcosgabrielgomes110-collab/Pybase2/discussions) ou issue.
